# 产品需求文档 (PRD)

## 文档信息

- **项目名称**: Web PPT（网页版PPT开发平台）
- **版本**: v2.0
- **创建日期**: 2026-03-06
- **对标物**: Reveal.js, Slidev, Marp, Spectacle
- **状态**: 草稿

---

## 1. 项目概述

### 1.1 背景与目标

**背景**: 用户需要一个基于Web技术的演示文稿开发平台，能够创建各类PPT模板、存放成品、资料管理。当前项目已实现基础框架（基于Reveal.js），但功能较为单一，需要扩展为完整的演示文稿管理系统。

**目标**: 构建一个功能完善的网页版PPT开发平台，支持模板管理、成品展示、素材管理、协作编辑等能力。

**对标参考**: 本产品参考 Reveal.js、Slidev、Marp、Spectacle，主要差异为：
- 增量：模板系统、素材管理、协作功能
- 修改：提供可视化编辑界面，降低使用门槛
- 减量：不追求复杂的动画特效，聚焦核心演示功能

### 1.2 目标用户

| 用户类型 | 描述 | 核心需求 |
|---------|------|---------|
| 开发者 | 技术演讲者、工程师 | 代码高亮、Markdown支持、快速创建 |
| 设计师 | 创意展示需求 | 丰富的模板、个性化定制 |
| 商务 | 商务演示需求 | 简洁模板、专业效果 |
| 教育者 | 课程演示需求 | 讲者备注、多媒体支持 |

### 1.3 核心价值主张

一个易于使用、功能完善的网页版PPT开发平台，让用户能够快速创建专业演示文稿。

### 2.2 用户故事

#### US-001: 模板创建与管理

**格式**: 作为用户，我想创建自定义模板，以便快速创建符合需求的演示文稿。

**详细描述**:
- **角色**: 所有用户
- **功能**: 创建新模板、编辑模板内容、复制模板、删除模板
- **价值**: 提高演示文稿创建效率，保持风格一致性
- **对标参考**: Slidev使用Markdown+frontmatter配置，参考其配置化思路

**验收标准**:
- AC-001-01: 用户可以创建新模板并保存
- AC-001-02: 用户可以编辑已有模板内容
- AC-001-03: 用户可以复制现有模板创建新模板
- AC-001-04: 用户可以删除不需要的模板

**边界条件**:
- 输入: 模板名称、描述、内容（HTML/Markdown）
- 输出: 成功保存到templates目录
- 异常: 名称重复时提示用户重命名

#### US-002: 实时预览

**格式**: 作为用户，我想在编辑时实时预览幻灯片效果，以便快速调整内容。

**详细描述**:
- **角色**: 所有用户
- **功能**: 编辑内容即时渲染、切换幻灯片预览
- **价值**: 所见即所得，提升编辑效率
- **对标参考**: Slidev的dev模式实时预览

**验收标准**:
- AC-002-01: 编辑Markdown内容后1秒内完成渲染
- AC-002-02: 支持分屏预览（编辑区+预览区）
- AC-002-03: 支持全屏预览模式

**边界条件**:
- 输入: Markdown/HTML内容
- 输出: 渲染后的幻灯片
- 异常: 语法错误时显示错误提示而非崩溃

#### US-003: 素材管理

**格式**: 作为用户，我想上传和管理演示素材，以便在幻灯片中快速使用。

**详细描述**:
- **角色**: 设计师、开发者
- **功能**: 上传图片、字体文件，管理素材库
- **价值**: 素材复用，避免重复上传
- **对标参考**: 竞品均无此功能，是我们的差异化点

**验收标准**:
- AC-003-01: 支持拖拽上传图片
- AC-003-02: 支持批量上传
- AC-003-03: 素材库展示缩略图
- AC-003-04: 支持插入素材到当前幻灯片

**边界条件**:
- 输入: 图片文件（jpg/png/svg/gif）
- 输出: 上传到assets目录，返回引用路径
- 异常: 文件过大时提示压缩

#### US-004: 主题系统

**格式**: 作为用户，我想切换演示文稿主题，以便快速改变视觉效果。

**详细描述**:
- **角色**: 设计师、商务用户
- **功能**: 选择预置主题、自定义颜色/字体/背景
- **价值**: 快速切换风格，满足不同场景
- **对标参考**: Reveal.js的theme系统 + Slidev的theme配置

**验收标准**:
- AC-004-01: 提供至少5种预置主题
- AC-004-02: 用户可自定义主题颜色
- AC-004-03: 主题切换即时生效

**边界条件**:
- 输入: 主题选择/自定义配置
- 输出: 应用主题后的幻灯片样式
- 异常: 自定义配置冲突时使用默认值

#### US-005: 讲者模式

**格式**: 作为演讲者，我想使用讲者模式进行演示，以便更好地控制演讲过程。

**详细描述**:
- **角色**: 演讲者、教育者
- **功能**: 演讲者视图（当前页+下一页+计时器+备注）
- **价值**: 专业演讲体验
- **对标参考**: Reveal.js的speaker view

**验收标准**:
- AC-005-01: 支持演讲者视图窗口
- AC-005-02: 显示当前页和下一页预览
- AC-005-03: 演讲计时器功能
- AC-005-04: 显示当前幻灯片备注

**边界条件**:
- 输入: 演示模式启动
- 输出: 讲者视图窗口
- 异常: 多显示器环境支持

#### US-006: 导出功能

**格式**: 作为用户，我想导出演示文稿，以便分享给他人或离线使用。

**详细描述**:
- **角色**: 所有用户
- **功能**: 导出为PDF、单个HTML文件、图片集
- **价值**: 方便分享和归档
- **对标参考**: Reveal.js的PDF导出、Marp的PDF导出

**验收标准**:
- AC-006-01: 一键导出为PDF（打印优化）
- AC-006-02: 导出为独立HTML（可离线播放）
- AC-006-03: 导出为PNG图片集

**边界条件**:
- 输入: 导出格式选择
- 输出: 对应格式的文件
- 异常: 导出大文件时显示进度

## 5. 竞品分析附录

### 5.1 竞品功能对比

| 维度 | Reveal.js | Slidev | Marp | Spectacle | 我们(v2.0) |
|-----|-----------|--------|------|-----------|-----------|
| **核心特性** |
| HTML演示 | ✓ | ✓ | ✓ | ✓ | ✓ |
| Markdown支持 | ✓ | ✓ | ✓ | ✗ | ✓ |
| 实时预览 | ✗ | ✓ | ✗ | ✓ | ✓ |
| 代码高亮 | ✓ | ✓ | ✓ | ✓ | ✓ |
| 模板系统 | ✗ | ✓ | ✗ | ✗ | ✓ |
| 素材管理 | ✗ | ✗ | ✗ | ✗ | ✓ |
| 主题系统 | ✓ | ✓ | ✗ | ✓ | ✓ |
| 讲者模式 | ✓ | ✓ | ✗ | ✓ | ✓ |
| 导出PDF | ✓ | ✓ | ✓ | ✓ | ✓ |
| 导出HTML | ✓ | ✓ | ✓ | ✗ | ✓ |
| **技术特点** |
| 技术栈 | 纯JS | Vue+Vite | CLI | React | Vue+Vite |
| 学习曲线 | 低 | 中 | 中 | 高 | 低 |
| 定制性 | 高 | 高 | 中 | 高 | 高 |
| 社区活跃 | 高 | 高 | 中 | 中 | 新建 |

### 5.2 最佳实践建议

#### 推荐采用

1. **Markdown驱动**: Slidev和Marp都采用Markdown，验证了其高效性
2. **实时预览**: Slidev的dev模式是杀手级功能
3. **配置化主题**: Reveal.js的CSS变量方案值得借鉴

#### 建议避免

1. **过度复杂的动画系统**: 竞品功能冗余，实际使用率低
2. **强制依赖Node.js**: 保证CDN可用性，降低门槛

#### 创新机会

1. **素材管理**: 竞品均无，是我们的差异化亮点
2. **模板市场**: 可扩展为社区生态
